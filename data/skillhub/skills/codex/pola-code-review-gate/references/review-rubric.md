# Pola Review Rubric

## 正确性

- 需求验收项是否都能被当前实现满足。
- 新逻辑是否处理空值、失败、超时、重复请求和异常返回。
- 数据结构和接口字段是否与调用方一致。

## 安全

- 不输出密钥和敏感数据。
- 权限校验不只放在前端。
- 文件上传、路径、URL 和外部输入需要校验。

## 可维护性

- 复用项目已有抽象。
- 新抽象有明确收益。
- 避免全局副作用和隐式依赖。

## 测试

- 单测覆盖核心分支。
- 集成测试覆盖真实调用边界。
- 回归测试覆盖本次修复的原始失败场景。

## 发布

- 环境变量、数据迁移、部署脚本和回滚点明确。
- 用户可见变更有 release 或 devlog 记录。
