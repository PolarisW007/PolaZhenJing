# OneClub 项目管理器

为OneClub跨境电商AI客户项目执行两大核心任务：
1. **生成/更新 project-log.md** — 标准化四阶段项目跟进日志
2. **整理项目目录** — 按阶段分类归档所有文件

## 项目根目录

```
/Users/wangchang/Desktop/王畅的AI工作/工作2026/Oneclub跟进/
```

## 标准目录结构

每个项目文件夹（S1–S9+）必须遵循：

```
S{N}.{客户城市}{客户名}/
├── {客户简称}-project-log.md    ← 根目录，唯一管理入口
├── 01-需求诊断/                 ← 阶段一文件
├── 02-系统设计/                 ← 阶段二文件
├── 03-实施交付/                 ← 阶段三文件
└── 04-持续运营/                 ← 阶段四文件
```

## 工作流程

### 任务A：生成新 project-log.md

```
1. 列出目标项目文件夹内所有文件
2. 读取核心文件提取元数据：
   - *+AI需求分析.md / *悟空AI机会点分析*.md → 企业画像、规模、品牌、平台
   - *纪要*.md / *纪要*.docx → 参会人、决策、合作意向、后续计划
   - *AI机会深度研究报告*.md → 行业分析、竞争格局
3. 读取 Oneclub/OneClub客户项目跟进管理表.csv → 客户优先级、评分、服务模式
4. 填充模板（见下方）生成 {客户简称}-project-log.md
5. 放置在项目文件夹根目录
```

### 任务B：更新已有 project-log.md

```
1. 读取当前 project-log.md
2. 识别新增文件（不在产出物列表中的）
3. 扫描新文件内容，提取关键进展
4. 在「文档更新记录」区域追加新条目（逆序，最新在上）
5. 更新相关步骤状态（⬜→🟡→✅）
6. 更新风险看板和超期追踪
7. 更新 Meta 中的「当前阶段」和「下次跟进」
```

### 任务C：整理目录结构

```
1. 创建四个阶段子目录（如不存在）：01-需求诊断/ 02-系统设计/ 03-实施交付/ 04-持续运营/
2. 按文件类型归档：
   - *需求分析*.md/pdf、*AI机会点*.md/pdf、*纪要*、会议录音.mp3/.docx → 01-需求诊断/
   - *架构*、*设计*、*技术选型*、*方案* → 02-系统设计/
   - *实施*、*交付*、*上线*、*测试* → 03-实施交付/
   - *运营*、*迭代*、*优化*、*报告* → 04-持续运营/
3. project-log.md 始终留在根目录，不移动
4. 移动后更新 project-log.md 中的文件路径引用
```

## 文档更新条目格式

每次更新必须包含以下结构（逆序追加）：

```markdown
#### 📅 {YYYY-MM-DD} ｜ 第 N 次更新（{摘要标题}）

**关键进展**
- ✅ {已完成事项1}
- ✅ {已完成事项2}

**🎯 当前最核心突破点**
> {单段描述当前最有价值的进展/决策/信号}

**🚧 当前最核心卡点（风险点）**
> ⚠️ **[高/中/低风险]** {具体描述}
> → 应对：{措施}
> → {升级条件，必须含时间约束}

**下一步Action**
- [ ] {具体行动}（责任人：XX，截止：MM-DD）
```

## 状态标记规范

| 标记 | 含义 | 使用场景 |
|------|------|---------|
| ✅ Done | 已完成 | 步骤完成、产出物交付 |
| 🟡 进行中 | 进行中 | 已启动但未完成 |
| ⬜ 待启动 | 未开始 | 排队等待 |
| 🔵需求诊断 | 阶段一 | Meta中当前阶段 |
| 🟢系统设计 | 阶段二 | Meta中当前阶段 |
| 🟠实施交付 | 阶段三 | Meta中当前阶段 |
| 🔴持续运营 | 阶段四 | Meta中当前阶段 |

## 注意事项

- 所有输出使用中文
- 渠道合作类客户（如S3雨果、S7辉煌明天）使用精简模板（无阶段二~四技术交付）
- S级客户（如S1桑椹、S8千岸）需特别关注合规风险和IPO敏感点
- 更新日志中突破点和卡点为**必填项**，不可省略
- 风险升级条件必须包含时间约束（如"1周内未收到→升级"）
